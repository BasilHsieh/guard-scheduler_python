# 開發日誌（DEVLOG）

## 專案背景

- 目標：把每月手工排班（約 1 小時）縮短到幾分鐘。
- 場景：6 名保全、平日 5 崗（A~E）、假日 2 崗（F/G）。
- 核心價值：不是做 SaaS，而是做一個可實際用、可展示產品思維的 side project。

---

## 2026-04-22：MVP 起版

### 1) 先定規格再寫程式

- 先把 6 條規則定義清楚，並區分：
  - 硬限制：規則 1~4
  - 公平目標：規則 5~6
- 補上 `carry_over`，避免跨月規則斷裂。

### 2) 排班引擎與匯出

- 新增 Python 求解器（回溯 + 多次嘗試挑最佳）。
- 新增 CSV / XLSX 匯出。
- 新增 JSON 驗證報告（違規 + 公平指標）。

### 3) 先有 CLI，再補 UI

- CLI 可直接產生單月班表與報告。
- 補上 Streamlit UI，讓非工程背景也能操作。

### 4) 啟動體驗修正

- 增加 `start.command`（Mac 雙擊啟動）。
- 修正 Streamlit 首次啟動 onboarding prompt 卡住問題。
- 新增 `.streamlit/config.toml` 統一啟動設定。

## 2026-04-23：功能深化與排錯

### 1) 日型改為 API 來源

- 串接 `api.pin-yi.me` 台灣行事曆 API。
- 以 `isHoliday` 作為上班/放假判定依據。
- CLI 保留 `--use-input-holidays` 作為 fallback。

### 2) 規則檢查「可解釋化」

- 原本只顯示「通過/失敗」，不足以做證據展示。
- 新增 `audits`：
  - 每條規則的門檻、實測值、是否通過、證據細節。
- UI 新增「規則稽核證據」區塊（可展開查看細項）。

### 3) 文案在地化

- 將規則檢查摘要改為一般使用者可讀的台灣中文。
- 避免英文化欄位（如 `repeat_count`、`checked_pairs`）。

### 4) 調班功能 v1

- 定義調班為「借班 + 還班」：
  - 借班日：原值班人休，代班人上
  - 還班日：代班人休，原值班人上
- 套用後自動修復剩餘班表，維持整體規則。
- 新增變更摘要（調整格數、調整日期）。

### 5) 調班邏輯修正

- 修正同工時限制（10h 對 10h / 12h 對 12h）。
- UI 還班日候選先做工時過濾，避免先選錯再報錯。
- 無解訊息改中文並加可行還班日建議。

### 6) 日期規則修正

- 原設計：還班日一定晚於借班日（造成月底易無解）。
- 調整為：
  - 還班日可早於或晚於借班日（同月內）
  - 但兩日期都必須是「今天與未來」。
- 新增「測試模式：自訂今天日期」方便驗證。

### 7) 使用者回饋導向修正

- 修正文案歧義（「上一次成功」改為「本次結果」語意）。
- 將「還班候選為空」改為中性說明，避免誤導成系統錯誤。

### 8) UI/UX 介面重構（不改功能）

- 目標：功能維持不變，但讓一般使用者第一次打開就看得懂。
- 改動重點：
  - 重新分成四個操作區：班表總覽 / 規則與證據 / 調班 / 下載。
  - 新增 dashboard 首屏（狀態卡、工時差、哨點差）。
  - 側欄改成「必要設定 + 進階參數」層次，降低操作負擔。
  - 規則違規說明改為更口語的台灣中文（非工程術語）。
- 保證項目：
  - 排班演算法、調班演算法、輸出格式完全不變。
  - 僅做視覺與資訊架構調整。

### 9) UI/UX 二次迭代（單頁工作台）

- 使用者回饋：上一版仍像工程介面，分頁切換增加認知成本。
- 本次調整：
  - 取消 tabs，改為單頁資訊架構（上方操作列 + 中段班表/規則 + 下方調班）。
  - 移除「司法院」字樣，產品名稱改為中性 `Guard Scheduler 排班工具`。
  - 色彩改為中性淺底與單一主色，降低視覺噪音。
  - 右側規則面板改為「規則健康」卡片，先呈現可行性再看細節證據。
- 不變項：
  - 排班結果、調班邏輯、匯出內容與 API 串接皆維持原功能。

### 10) UI/UX 三次迭代（Reference 對齊）

- 依照新參考圖重整視覺骨架：
  - 新增左側導覽視覺（Dashboard / Schedule Matrix / Swap / Compliance）。
  - 主內容改成 `Schedule Matrix` + `Critical Rule Violations` 版型語意。
  - 上方加入產品工具列風格（Generate / Export）。
- 內容層次調整：
  - 先顯示矩陣與違規卡，再進入借班還班操作。
  - 規則健康、工時分布、哨點均衡放在右欄作為監控面板。
- 保持不變：
  - 演算法、調班求解、輸出格式、API 來源皆未改動。

### 11) 文件同步與交接整理

- 重寫 `README`，補上：
  - 啟動方式（`start.command` 與手動啟動）
  - 一般使用者操作流程
  - 文件索引與目前限制
- PRD 補充 UX 原則與目標資訊架構（產品語言）
- TECH SPEC 補充目前單頁 UI 架構與操作區責任分工（技術語言）
