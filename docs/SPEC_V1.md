# 保全排班工具 Spec v1

## 1. 目標

建立一個輕量工具，將每月手工 Excel 排班（約 1 小時）縮短到幾分鐘，並降低規則違反風險。

## 2. 範圍

### 2.1 In Scope (MVP)

- 6 名保全人員月排班
- 固定哨點與時數設定
- 自動排班
- 規則檢查報告
- 匯出 CSV / XLSX

### 2.2 Out of Scope (v1 不做)

- 多組織多租戶
- 帳號權限系統
- 排班申請流程（請假、審核）
- 自動同步外部 HR 系統

## 3. 核心資料模型

### 3.1 人員

- `id`: string
- `name`: string

### 3.2 哨點

- 平日：`A/B/C` (10h), `D/E` (12h)
- 假日：`F/G` (12h)

### 3.3 每日需求

- 平日：`A~E` 共 5 人
- 假日：`F/G` 共 2 人

### 3.4 跨月狀態（carry_over）

每人可帶入：
- `consecutive_days`: 上月月底連續上班天數
- `last_post`: 上次班別（避免次日同哨點）
- `last_holiday_dow`: 上次假日班是六或日
- `last_holiday_post`: 上次假日哨點 `F/G`

## 4. 規則定義

### 4.1 硬限制（Hard Constraints）

1. 連續上班不得超過 6 天  
2. 同一哨點不得連兩天  
3. 假日班星期需六日交替（不可連續兩個六或兩個日）  
4. 假日哨點需 F/G 交替

### 4.2 軟限制（Soft Objectives）

5. 月總工時公平：`max(hours) - min(hours) <= 12` 目標  
6. 同哨點分配公平：每個哨點在人員間次數差距 `<= 1` 目標

> 設為軟限制可避免特定月份因休假結構導致「無解」。

## 5. 排班演算法（v1）

### Phase A：配額目標

先計算每個哨點在該月總需求，分配給每位人員的目標次數：
- 基礎配額：`floor(total_post_slots / guard_count)`
- 餘數配額：優先補給目前預估時數較低者

### Phase B：逐日最佳化

對每日所需哨點，從可行人員中用回溯搜尋挑選當日最佳組合：
- 先過濾硬限制不合法候選
- 以「配額缺口 + 時數平衡 + 連勤懲罰」計分
- 多次嘗試（randomized restarts）挑選最佳整月解

## 6. 輸入/輸出

### 6.1 輸入

- 月份（年、月）
- 人員名單
- 行事曆來源（二選一）
  - 台灣行事曆 API（`isHoliday` 判定）
  - 手動 holidays 清單（fallback）
- carry_over（可選）

### 6.2 輸出

- `schedule_YYYY_MM.csv`
- `schedule_YYYY_MM.xlsx`
- `report_YYYY_MM.json`（含違規與公平指標）

## 7. 驗收標準（MVP）

- 所有硬限制違規數為 0
- 工時差距可計算並可追蹤（目標 <= 12h）
- 每哨點分配差距可計算並可追蹤（目標 <= 1）
- 可在 5 秒內完成單月排班（一般筆電）
- 能成功匯出 CSV 與 XLSX
